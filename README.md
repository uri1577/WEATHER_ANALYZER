WEATHER_ANALYZER_OpenMeteo.ubi
Подробное описание блока и внесенных поправок

Дата: 2026-07-01

1. Назначение блока

Блок WEATHER_ANALYZER рассчитывает вероятность осадков на ближайшие 6 часов.

Он использует:
- внешний прогноз Open-Meteo;
- локальные данные BME280;
- необязательный локальный датчик дождя;
- кольцевую память на 36 записей;
- уже подключенный Wi-Fi контроллера.

Блок не управляет подключением Wi-Fi:
- не задает SSID;
- не задает пароль;
- не вызывает WiFi.begin;
- не выполняет переподключение к сети.

Блок только проверяет, что Wi-Fi уже подключен:

WiFi.status() == WL_CONNECTED

Если Wi-Fi подключен, блок делает запрос к Open-Meteo.
Если Wi-Fi не подключен или сервер недоступен, блок продолжает работать по последним данным, BME280 и истории.


2. Основные входы блока

WA_EN : Bool

Назначение:
Включение или выключение блока.

Градация:
- false: блок выключен, OUT_HOUR = 1, OUT_PROB = 0;
- true: блок выполняет расчет.


LATITUDE : Float

Назначение:
Широта точки прогноза для Open-Meteo.

Пример:
50.4501


LONGITUDE : Float

Назначение:
Долгота точки прогноза для Open-Meteo.

Пример:
30.5234


LOCAL_TEMP : Float

Назначение:
Температура с BME280 в градусах Цельсия.

Градация проверки:
- меньше -50: значение считается ошибочным;
- от -50 до 80: используется в локальном анализе;
- больше 80: значение считается ошибочным.

Если значение ошибочное, локальная часть BME280 временно отключается.


LOCAL_HUMIDITY : Float

Назначение:
Влажность с BME280 в процентах.

Градация проверки:
- меньше 0: значение считается ошибочным;
- 0..49: сухо, вероятность осадков уменьшается;
- 50..69: нейтральная зона;
- 70..79 и влажность растет: слабый плюс к вероятности;
- 80..89 и влажность растет: средний плюс;
- 90..100 и влажность растет: сильный плюс;
- больше 100: значение считается ошибочным.


LOCAL_PRESSURE : Float

Назначение:
Давление с BME280.

Ожидаемые единицы:
- hPa;
- mbar.

Дополнительно:
Если подано давление в Pa, примерно 80000..110000, блок делит его на 100.

Градация проверки:
- меньше 800 hPa: значение считается ошибочным;
- 800..1100 hPa: используется в расчете;
- больше 1100 hPa: значение считается ошибочным.


RAIN_SENSOR_USE : Bool

Назначение:
Разрешение учитывать локальный датчик дождя.

Градация:
- false: датчик дождя не используется, RAIN_SENSOR_STATE игнорируется;
- true: датчик используется.


RAIN_SENSOR_STATE : Bool

Назначение:
Состояние локального датчика дождя.

Градация:
- false: датчик не подтверждает дождь;
- true при RAIN_SENSOR_USE = true: текущие осадки локально подтверждены.


3. Параметр блока

REQUEST_INTERVAL_MIN : Int

Назначение:
Интервал запроса Open-Meteo в минутах.

Градация:
- меньше 10: принудительно используется 10 минут;
- 10..360: используется заданное значение;
- больше 360: принудительно используется 360 минут.

Значение по умолчанию:
60 минут.


4. Выходы блока

OUT_HOUR : Int

Назначение:
Текущий отображаемый час прогноза.

Градация:
- 1: прогноз на 1 час вперед;
- 2: прогноз на 2 часа вперед;
- 3: прогноз на 3 часа вперед;
- 4: прогноз на 4 часа вперед;
- 5: прогноз на 5 часов вперед;
- 6: прогноз на 6 часов вперед.

Цикл отображения:
- 1H показывается 20 секунд;
- 2H показывается 20 секунд;
- 3H показывается 20 секунд;
- 4H показывается 20 секунд;
- 5H показывается 20 секунд;
- 6H показывается 20 секунд;
- затем снова 1H.

Полный цикл:
120 секунд.


OUT_PROB : Int

Назначение:
Вероятность осадков для текущего OUT_HOUR.

Градация:
- 0..20: осадки маловероятны;
- 21..40: низкая вероятность;
- 41..60: средняя вероятность;
- 61..80: высокая вероятность;
- 81..100: очень высокая вероятность.

Пример:
OUT_HOUR = 3
OUT_PROB = 65

Значит:
3H = 65%


5. Работа с Open-Meteo

Текущая реализация делает HTTP-запрос через WiFiClient.

Используемый хост:
api.open-meteo.com

Порт:
80

Почему используется HTTP, а не HTTPS:
Чтобы не требовать WiFiClientSecure и не зависеть от include-библиотек, которые FLProg не добавляет автоматически из пользовательского блока.

Запрашиваемые hourly-поля:
- temperature_2m;
- relative_humidity_2m;
- surface_pressure;
- precipitation_probability;
- precipitation;
- rain;
- showers;
- snowfall;
- weather_code.

Используется:
- forecast_hours=12;
- timezone=auto.

В расчет берутся первые 6 часов.

Если Open-Meteo возвращает precipitation_probability, блок использует его как базовую вероятность.

Если precipitation_probability отсутствует, используется запасной расчет:
- precipitation > 0 добавляет признак осадков;
- rain > 0 добавляет признак дождя;
- showers > 0 добавляет признак ливней;
- snowfall > 0 добавляет признак снега;
- weather_code из группы осадков добавляет признак осадков.


6. Локальный анализ BME280

Блок анализирует не только текущие значения, но и изменение относительно предыдущей записи.

PRESSURE_DELTA = LOCAL_PRESSURE - PREVIOUS_PRESSURE

Градация давления:
- PRESSURE_DELTA <= -1.5 hPa: сильный рост вероятности;
- PRESSURE_DELTA <= -1.0 hPa: заметный рост вероятности;
- PRESSURE_DELTA <= -0.5 hPa: небольшой рост вероятности;
- PRESSURE_DELTA >= +0.8 hPa: снижение вероятности;
- PRESSURE_DELTA >= +1.5 hPa: сильное снижение вероятности.


HUMIDITY_DELTA = LOCAL_HUMIDITY - PREVIOUS_HUMIDITY

Градация влажности:
- влажность >= 90% и растет: сильный плюс;
- влажность >= 80% и растет: средний плюс;
- влажность >= 70% и растет: слабый плюс;
- влажность < 50%: минус;
- влажность падает: минус.


TEMP_DELTA = LOCAL_TEMP - PREVIOUS_TEMP

Градация температуры:
- температура заметно падает: плюс;
- температура резко падает: сильный плюс;
- температура растет, а влажность падает: минус.


7. Точка росы

Используется упрощенная формула:

DEW_POINT = LOCAL_TEMP - ((100 - LOCAL_HUMIDITY) / 5)
DEW_DIFF  = LOCAL_TEMP - DEW_POINT

Градация:
- DEW_DIFF <= 1: воздух почти насыщен влагой, сильный плюс;
- DEW_DIFF <= 2: заметный плюс;
- DEW_DIFF <= 4: слабый плюс;
- DEW_DIFF > 8: минус.


8. Датчик дождя

Если:
RAIN_SENSOR_USE = true
RAIN_SENSOR_STATE = true

Тогда:
- текущие осадки считаются локально подтвержденными;
- P_FINAL[1] не ниже 90;
- P_FINAL[2] не ниже 70;
- P_FINAL[3..6] не становятся автоматически 100.

Причина:
Датчик дождя подтверждает текущие осадки, но не гарантирует дождь на весь будущий 6-часовой период.

Если:
RAIN_SENSOR_USE = false

Тогда:
- блок не имеет права считать дождь абсолютно подтвержденным;
- используется только вероятностная оценка.


9. Кольцевая память

Размер:
36 записей.

Шаг записи:
10 минут.

Глубина:
6 часов.

Запись идет по кругу:
0 -> 1 -> 2 -> ... -> 35 -> 0

В записи сохраняются:
- температура;
- влажность;
- давление;
- изменение давления;
- изменение влажности;
- изменение температуры;
- близость точки росы;
- текущий час;
- текущий месяц;
- подтверждение осадков;
- источник подтверждения;
- рассчитанные вероятности для 1..6 часов.

Назначение памяти:
- историческая поправка;
- поиск похожих условий;
- усиление или ослабление прогноза.


10. Итоговый расчет

Для каждого часа 1..6:

P_FINAL =
Open-Meteo * W_OPEN
+ Local BME280 * W_LOCAL
+ History * W_HISTORY
+ season correction
+ daytime correction

Базовые веса:
- Open-Meteo = 0.40;
- BME280 = 0.40;
- History = 0.20.

Если BME280 ошибочный:
- W_LOCAL = 0;
- вес Open-Meteo повышается;
- история продолжает использоваться.

Если Open-Meteo недоступен:
- расчет идет по BME280 и истории;
- используются последние сохраненные данные, если они есть.


11. Защита от ложных срабатываний

Роса или туман:
Если влажность высокая, давление не падает, Open-Meteo не показывает осадки, а сейчас ночь или утро, вероятность уменьшается.

Локальный скачок влажности:
Если влажность резко выросла, давление стабильно, температура стабильна, а Open-Meteo не показывает осадки, вклад влажности ограничивается.

Ошибка BME280:
Если температура, влажность или давление выходят за допустимые пределы, локальная часть временно отключается.


12. Сезонная поправка

Зима:
При температуре около 0 градусов или ниже вероятность немного повышается.

Весна:
При падении давления добавляется слабая сезонная поправка.

Лето:
Высокая температура и высокая влажность повышают вероятность.

Осень:
Высокая влажность и падение давления дают более сильную поправку.


13. Поправка времени суток

Утро:
Высокая влажность без падения давления чаще считается росой или туманом, вероятность снижается.

День:
Летом высокая температура и высокая влажность повышают вероятность.

Вечер:
Падение температуры и высокая влажность немного повышают вероятность.

Ночь:
Высокая влажность без падения давления снижает вероятность, чтобы не спутать дождь с росой или туманом.


14. Внесенные поправки упаковки и компиляции

Поправка 1. Кодировка файла

Проблема:
FLProg не открывал файл и показывал Read Error.

Причина:
Исходный образец был сохранен как UTF-16 LE с BOM, а первый собранный файл был UTF-8.

Исправление:
Файл WEATHER_ANALYZER_OpenMeteo.ubi перекодирован в UTF-16 LE с BOM.


Поправка 2. Служебные идентификаторы sixx.id

Проблема:
Упаковка отличалась от образца.

Исправление:
sixx.id перенумерованы последовательно, ссылки sixx.idref обновлены.


Поправка 3. Падение FLProg при переносе блока на визуальную область

Проблема:
FLProg зависал и закрывался при переносе блока на рабочую область.

Причина из error.log:
CodeUserBlockDeclareCodePart>>isCorrectWithProject:
получал Text вместо OrderedCollection.

FLProg пытался перебрать текст #include посимвольно и вызвать у символа метод isCorrectWithProject.
На символе # происходило падение:
Character doesNotUnderstand: isCorrectWithProject:

Исправление:
declareCodePart сделан как в образце:
code = OrderedCollection.

Логика расчета перенесена только в loopCodePart.


Поправка 4. Ошибка компиляции WiFiClientSecure / HTTPClient / ArduinoJson

Проблема:
После исправления declareCodePart компилятор не видел:
- WiFiClientSecure;
- HTTPClient;
- HTTP_CODE_OK;
- DynamicJsonDocument;
- DeserializationError;
- JsonObject.

Причина:
FLProg не добавил include-файлы:
- WiFiClientSecure.h;
- HTTPClient.h;
- ArduinoJson.h.

Исправление:
Зависимости от этих классов удалены.

Текущая реализация использует:
- WiFiClient;
- обычный HTTP порт 80;
- ручной разбор нужных массивов ответа Open-Meteo.

Это сделано для совместимости с пользовательским блоком FLProg, где безопасное добавление include через declareCodePart приводит к падению визуального редактора.

17. Новый выход W2_OM_STATUS

Тип:
Bool.

Назначение:
Показывает статус получения и разбора данных Open-Meteo.

Градация:
- false: данных Open-Meteo еще нет, Wi-Fi не подключен, запрос не выполнен, запрос не удался или ответ не разобран;
- true: последний выполненный запрос Open-Meteo успешно получен и разобран.

Особенность:
Выход не является текстовым статусом. Он специально сделан как Bool для последующей передачи через ESP-NOW.

Поведение:
- при W2_EN = false выход W2_OM_STATUS = false;
- перед новым запросом статус сбрасывается в false;
- после успешного получения и разбора ответа Open-Meteo статус становится true;
- если наступило время запроса, но Wi-Fi не подключен, статус становится false;
- между запросами сохраняется результат последнего успешного/неуспешного состояния.


18. Переименование для исключения наложения V1 и V2

Чтобы V2 не накладывался на уже импортированный V1:
- имя блока изменено с WEATHER_ANALYZER на WEATHER_ANALYZER_V2;
- имена входов и выходов переведены на префикс W2_;

- параметр REQUEST_INTERVAL_MIN переименован в W2_REQ_MIN;
- внутренние UUID-идентификаторы портов, параметров и блока обновлены.

Примеры переименования:
- WA_EN -> W2_EN;
- LATITUDE -> W2_LAT;
- LONGITUDE -> W2_LON;
- LOCAL_TEMP -> W2_TEMP_C;
- LOCAL_HUMIDITY -> W2_HUM_PCT;
- LOCAL_PRESSURE -> W2_PRESS_HPA;
